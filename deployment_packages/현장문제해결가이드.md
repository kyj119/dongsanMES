## 🔴 문제 1: localhost:5285 페이지가 안 열려요!

### 증상
- 웹 브라우저에서 `http://localhost:5285` 입력 시 "페이지를 표시할 수 없습니다" 오류
- 또는 "연결할 수 없습니다" 오류

### 해결 방법

#### Step 1: MES 서버가 실행 중인지 확인
```powershell
# PowerShell 또는 CMD 열기
cd C:\dongsanMES\MESSystem

# 서버 실행
dotnet run
```

**실행 성공 시 콘솔 출력 예시**:
```
info: Microsoft.Hosting.Lifetime[14]
      Now listening on: http://0.0.0.0:5285
info: Microsoft.Hosting.Lifetime[0]
      Application started. Press Ctrl+C to shut down.
```

이 메시지가 보이면 ✅ **정상**입니다!

#### Step 2: 브라우저에서 접속 테스트

**테스트할 URL들** (순서대로 시도):
1. `http://localhost:5285`
2. `http://127.0.0.1:5285`
3. `http://[컴퓨터IP]:5285` (예: `http://192.168.0.100:5285`)

#### Step 3: 방화벽 확인

서버가 실행 중인데도 접속 안 되면:

1. **Windows Defender 방화벽 열기**
   - 시작 > "방화벽" 검색
   - "고급 보안이 포함된 Windows Defender 방화벽"

2. **인바운드 규칙** 클릭

3. **새 규칙** 클릭:
   - 규칙 유형: **포트**
   - 프로토콜: **TCP**
   - 특정 로컬 포트: **5285**
   - 작업: **연결 허용**
   - 이름: **MES Server Port 5285**

#### Step 4: 포트 충돌 확인

다른 프로그램이 5285 포트를 사용 중일 수 있습니다:

```powershell
# 포트 사용 확인
netstat -ano | findstr :5285

# 출력 예시:
# TCP    0.0.0.0:5285    0.0.0.0:0    LISTENING    12345
#                                                   ↑ PID
```

만약 다른 프로그램이 사용 중이면:
- 해당 프로그램 종료
- 또는 `launchSettings.json`에서 포트 변경 (5000, 5001, 8080 등)

#### Step 5: HTTPS 리다이렉션 문제

`dotnet run` 실행 시 자동으로 HTTPS로 리다이렉트될 수 있습니다.

**해결**: 환경 변수 설정
```powershell
# PowerShell에서
$env:ASPNETCORE_URLS = "http://localhost:5285"
dotnet run
```

또는 **launchSettings.json 수정**:
```json
"profiles": {
  "http": {
    "commandName": "Project",
    "launchBrowser": true,
    "applicationUrl": "http://localhost:5285",
    "environmentVariables": {
      "ASPNETCORE_ENVIRONMENT": "Development"
    }
  }
}
```

#### Step 6: 빌드 확인

프로젝트가 제대로 빌드되었는지 확인:
```powershell
cd C:\dongsanMES\MESSystem
dotnet build

# 에러 없이 "Build succeeded" 나와야 함
```

## 🔴 문제 2: install_collector.bat 실행해도 아무 일도 안 일어나요!

### 증상
- `install_collector.bat` 우클릭 → 관리자 권한으로 실행
- CMD 창이 1초 깜빡하고 사라짐
- 또는 아무것도 안 보임

### 해결 방법

#### Step 1: CMD에서 직접 실행 (추천!)

배치 파일이 빠르게 종료되어 오류 메시지를 못 볼 수 있습니다.

```powershell
# 1. CMD를 관리자 권한으로 실행
#    시작 > "cmd" 검색 > 우클릭 > "관리자 권한으로 실행"

# 2. 배치 파일이 있는 폴더로 이동
cd C:\dongsanMES

# 3. 배치 파일 실행
install_collector.bat

# 이제 에러 메시지를 볼 수 있습니다!
```

#### Step 2: .NET 8.0 Runtime 설치 확인

가장 흔한 원인: **.NET 8.0이 설치되지 않음**

**확인 방법**:
```powershell
dotnet --version
```

**결과**:
- ✅ `8.0.x` → 정상
- ❌ `'dotnet'은(는) 내부 또는 외부 명령...` → .NET 미설치

**설치 방법**:
1. https://dotnet.microsoft.com/download/dotnet/8.0
2. **ASP.NET Core Runtime 8.0.x - Windows x64 Hosting Bundle** 다운로드
3. 설치 후 PC 재부팅
4. 다시 `dotnet --version` 확인

#### Step 3: 경로 확인

배치 파일이 실행되려면 **MESCollector 소스 파일이 필요합니다**.

**필요한 파일 구조**:
```
C:\dongsanMES\
├── install_collector.bat       ← 이 파일 실행
└── MESCollector/                ← ⚠️ 이 폴더가 있어야 함!
    ├── Program.cs
    ├── MESCollector.csproj
    ├── appsettings.json
    ├── Models/
    └── Services/
```

**MESCollector 폴더가 없다면**:
→ GitHub에서 다운로드: `https://github.com/kyj119/dongsanMES`
→ `MESCollector` 폴더 전체 복사

#### Step 4: 수동 설치 (배치 파일 없이)

배치 파일이 계속 안 되면 수동으로 설치하세요:

```powershell
# 1. 설치 폴더 생성
mkdir C:\MESCollector

# 2. TOPAZ_RIP 폴더 생성
mkdir C:\TOPAZ_RIP\preview
mkdir C:\TOPAZ_RIP\printlog
mkdir C:\TOPAZ_RIP\job

# 3. MESCollector 소스 복사
#    GitHub에서 다운로드한 MESCollector 폴더를
#    C:\MESCollector\ 로 복사

# 4. 설정 파일 수정
notepad C:\MESCollector\appsettings.json

# 5. 테스트 실행
cd C:\MESCollector
dotnet run
```

#### Step 5: 관리자 권한 확인

**확인 방법**:
```powershell
# CMD를 관리자 권한으로 열었는지 확인
net session

# 결과:
# ✅ "로컬 영역 연결..." → 관리자 권한 O
# ❌ "액세스가 거부되었습니다." → 관리자 권한 X
```

## 🔴 문제 3: Collector 실행 시 "MES 서버에 연결할 수 없습니다" 오류

### 증상
```
[ERR] MES 서버에 연결할 수 없습니다: http://192.168.0.100:5000
[ERR] 재시도 중... (1/3)
```

### 해결 방법

#### Step 1: MES 서버가 실행 중인지 확인
```powershell
# 서버 PC에서
cd C:\dongsanMES\MESSystem
dotnet run

# "Now listening on: http://0.0.0.0:5285" 확인
```

#### Step 2: 네트워크 연결 확인
```powershell
# 출력기 PC에서 서버로 핑 테스트
ping 192.168.0.100

# 응답이 오면 네트워크 정상
```

#### Step 3: 방화벽 확인 (서버 PC)
- Windows Defender 방화벽에서 5285 포트 허용
- 회사 방화벽/라우터 확인

#### Step 4: appsettings.json 확인
```json
{
  "Collector": {
    "ServerUrl": "http://192.168.0.100:5285",  ← 포트 번호 확인!
    "CollectorId": "COLLECTOR-1F",
    ...
  }
}
```

## 🔴 문제 4: "카드를 찾을 수 없습니다" 오류

### 증상
Collector가 이벤트를 전송했지만 MES 서버에서 거부

### 해결 방법

1. **MES 웹에서 카드 먼저 생성**
   - 로그인: `http://서버IP:5285`
   - 주문서 작성
   - 카드 생성 (카드 번호 확인!)

2. **카드 번호 형식 확인**
   - 올바른 형식: `20260206-01-1`
   - 날짜-순번-항번

3. **파일명과 카드 번호 일치**
   - 파일명: `20260206-01-1.bmp.tsc`
   - 카드 번호: `20260206-01-1`
